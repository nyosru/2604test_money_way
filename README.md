# 🧪 Тестовое задание: Найди баг в продакшен-коде

**Вакансия:** PHP-разработчик (финансовые интеграции)  
**Компания:** amgroup  
**Время на выполнение:** 2-4 часа  
**Срок сдачи:** 5-7 дней

---

## 📋 Описание кейса

### Ситуация

Клиент обратился с проблемой в интеграции между Альфа-Банком и МойСклад.

### Что произошло

| Проблема | Детали |
|----------|--------|
| **200 платежей** | Привязались к **одному и тому же счёту** |
| **Ожидалось** | Каждый платёж → свой индивидуальный счёт |
| **Факт** | Все платежи созданы в одно время интеграцией |
| **Данные** | У платежей разное назначение и корректные номера счетов |

### Пример назначения платежа

```
"Оплата по сч/ф 1020 от 19.02.2025 по договору № Б/Н от 16.12.2024 за Закупка поломоечных машин ТР ЮГ в т.ч. НДС 40.487,50"
```

---

## 📁 Что у тебя есть

| Файл | Описание |
|------|----------|
| `SyncCommand.php` | Фрагмент класса синхронизации платежей |
| `PaymentsIn` | Входящий платёж МойСклад |
| `InvoiceOut` | Счёт покупателю в МойСклад |
| `MoyskladApp` | Клиент для доступа к API МойСклад |
| `ah` | Обёртка над массивом для удобной работы |

**Для решения проблемы достаточно двух методов:**
- `invoiceNumberInPurpose()` — поиск номера счёта в назначении платежа
- `attachToInvoiceOut()` — привязка платежей к счетам

---

## 🎯 Твоя задача

### 1. Найти причину
Выяснить, **по какой причине 200 разных платежей привязались к одному счёту**.

### 2. Исправить код
Внести изменения, чтобы **кейс больше не повторился**.

### 3. Рефакторинг
Улучшить читаемость и понятность кода.
